## 🏗️ ARCHITECTURE OVERVIEW

### Database Layer (Prisma ORM)
- **15 core tables**: Company, Facility, User, Role, Lead, Resident, Visit, Billing, Charting, Document, Employee, Timesheet, AuditLog, MedicaidConfig, IntegrationLog
- **Fully normalized** 3NF design
- **Soft deletes** for all records (never delete, just mark inactive)
- **Audit trails** on all tables (who, what, when)
- **Schema file**: `ARCHITECTURE.md` (complete Prisma schema included)

### API Layer (RESTful)
- **80+ endpoints** fully documented in OpenAPI 3.0
- **RBAC-enforced**: 8 roles, permission checks on every endpoint
- **Standard response format**: `{ success, data, error, status }`
- **Error handling**: Consistent error codes (400, 401, 403, 404, 500)
- **Pagination**: Offset-based (limit, offset, total)
- **Authentication**: Bearer token JWT in Authorization header

### Frontend Layer (React)
- **4 distinct portals**: Facility, Care Staff (embedded), Family, Super Admin
- **Component library**: 50+ reusable components (Button, Card, Form, Table, Modal, etc.)
- **Redux state**: Centralized company/facility/user context
- **Pages**: 30+ pages organized by feature
- **Shared UI**: Material-UI design system

### Security Layer (HIPAA-Critical!)
- **Data at rest**: AES-256 encryption for sensitive fields (SSN, medical records)
- **Data in transit**: HTTPS/TLS 1.3+
- **Authentication**: JWT (15-minute expiry) + refresh token (7 days)
- **Authorization**: RBAC checked on every API endpoint
- **Audit logging**: Every action logged (user_id, action, resource, timestamp)
- **Access control**: Users can only access data they're assigned to
- **Soft deletes**: Never delete records, mark as inactive

## 🛠️ DEVELOPMENT PATTERNS

### Backend API Pattern (Express.js)

**Anatomy of an endpoint**:
```typescript
// 1. Route (in routes/residents.ts)
router.post('/', requireAuth, requireRole('FACILITY_ADMIN'), createResident);

// 2. Controller (in controllers/residentController.ts)
export const createResident = async (req, res) => {
  try {
    const resident = await residentService.create(req.body, req.user.facilityId);
    
    // 3. Audit log
    await auditService.log({
      userId: req.user.id,
      action: 'CREATE_RESIDENT',
      resource: 'Resident',
      resourceId: resident.id,
    });
    
    res.status(201).json({
      success: true,
      data: resident,
      status: 'RESIDENT_CREATED',
    });
  } catch (error) {
    res.status(400).json({
      success: false,
      error: error.message,
      status: 'RESIDENT_CREATE_FAILED',
    });
  }
};

// 4. Service (in services/residentService.ts)
export const create = async (data, facilityId) => {
  // Validate input
  const validation = validateResident(data);
  if (!validation.valid) throw new Error(validation.error);
  
  // Check duplicates
  const existing = await checkDuplicate(data.ssn, facilityId);
  if (existing) throw new Error('Duplicate SSN found');
  
  // Create record
  const resident = await prisma.resident.create({
    data: {
      ...data,
      facilityId,
    },
  });
  
  return resident;
};

// 5. Tests (in tests/residents.test.ts)
describe('POST /residents', () => {
  it('should create resident with valid data', async () => {
    const res = await request(app)
      .post('/residents')
      .set('Authorization', `Bearer ${token}`)
      .send(validResidentData);
    
    expect(res.status).toBe(201);
    expect(res.body.success).toBe(true);
    expect(res.body.data.name).toBe(validResidentData.name);
  });
  
  it('should reject duplicate SSN', async () => {
    await prisma.resident.create({ data: existingResident });
    
    const res = await request(app)
      .post('/residents')
      .set('Authorization', `Bearer ${token}`)
      .send({ ...newResident, ssn: existingResident.ssn });
    
    expect(res.status).toBe(400);
    expect(res.body.error).toContain('Duplicate');
  });
});
```

### Frontend Component Pattern (React)

**Anatomy of a feature component**:
```typescript
// 1. Redux slice (in redux/slices/residentSlice.ts)
const residentSlice = createSlice({
  name: 'residents',
  initialState: { list: [], loading: false, error: null },
  reducers: { ... },
  extraReducers: (builder) => {
    builder
      .addCase(fetchResidents.pending, (state) => { state.loading = true; })
      .addCase(fetchResidents.fulfilled, (state, action) => {
        state.loading = false;
        state.list = action.payload;
      });
  },
});

export const fetchResidents = createAsyncThunk(
  'residents/fetch',
  async (filters) => await residentAPI.getList(filters),
);

// 2. API service (in services/api.ts)
export const residentAPI = {
  getList: (filters) => api.get('/residents', { params: filters }),
  create: (data) => api.post('/residents', data),
  update: (id, data) => api.put(`/residents/${id}`, data),
};

// 3. Page component (in pages/Residents.tsx)
export const ResidentsPage = () => {
  const dispatch = useDispatch();
  const { list, loading } = useSelector((state) => state.residents);
  const [filters, setFilters] = useState({});
  
  useEffect(() => {
    dispatch(fetchResidents(filters));
  }, [filters]);
  
  return (
    <Container>
      <PageHeader title="Residents" action={<CreateButton />} />
      <ResidentsTable residents={list} loading={loading} />
    </Container>
  );
};

// 4. Reusable component (in components/ResidentsTable.tsx)
export const ResidentsTable = ({ residents, loading }) => (
  <MaterialTable
    columns={[
      { name: 'name', title: 'Name' },
      { name: 'status', title: 'Status' },
    ]}
    data={residents}
    isLoading={loading}
  />
);

// 5. Tests (in tests/Residents.test.tsx)
describe('ResidentsPage', () => {
  it('should display list of residents', async () => {
    const { getByText } = render(<ResidentsPage />);
    
    await waitFor(() => {
      expect(getByText('John Doe')).toBeInTheDocument();
    });
  });
});
```
